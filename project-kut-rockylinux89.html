[한국기술교육대학][학사시스템] CentOS7.9 EOL로 인한 RockyOS 8.9 Apache/Tomcat/CLUSTER/APM 웹 시스템 재구축 프로젝트

2024.12.02 ~ 2025.01

Apache 2.4.33 Tomcat 8.5.54 Infinispan 7 Rocky Linux 8.9
개요
  • 인프라 환경 이관: CentOS 7.9 EOL에 대응하여 Rocky Linux 8.9 VM 서버 기반의 신규 인프라 환경 수립.
  • 고가용성(HA) 아키텍처 구축: 학사 트래픽 분산을 위해 WEB 4대 - WAS 4대 구조의 로드밸런싱 설정 및 Infinispan 7을 이용한 세션 클러스터링(2대) 서버 연동.
역할
  • 인프라 엔지니어(RockyOS 8.9 VM 서버에 WEB 4대, WAS 4대, CLUSTER 2대, APM 1대 학사 웹시스템 설치 및 구성)
성과
  • 보안 및 운영 안정성 확보: EOL 대상 OS 이관을 통해 보안 취약점 리스크 제거 및 지속 가능한 오픈소스 운영 기반 마련.
  • 무중단 가용성 확보: 세션 클러스터링 환경 정밀 튜닝을 통해 특정 서버 다운 시에도 학사 서비스가 정상 유지되도록 가용성 극대화.
개선점
  • WEB/WAS 환경 구축 초기에는 도입 대상 미들웨어(Apache/Tomcat) 자체의 버전 선정에 집중했습니다. 그러나 실제 구축 과정을 거치며 개별 버전을 넘어 OS → JDK → WAS → 애플리케이션으로 이어지는 전체 시스템 구성 요소 간의 종속성과 교차 호환성을 선제적으로 검증하는 것이 인프라 안정성의 핵심임을 체감했습니다. 반드시 사전에 종속성 및 교차 호환성 조사를 바탕으로 인프라 설계 시 시행착오를 최소화해야겠다고 느꼈습니다.
